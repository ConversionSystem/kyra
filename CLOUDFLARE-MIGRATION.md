## Known Limitations vs Vercel

### 1. Vercel Cron Jobs â†’ Not available on Workers
`vercel.json` had cron jobs for `/api/proactive/check` (every 3h) and `/api/reminders/check` (every 5min). These need to be replaced with:
- **Cloudflare Cron Triggers** â€” add to `wrangler.toml`:
  ```toml
  [triggers]
  crons = ["0 */3 * * *", "*/5 * * * *"]
  ```
  Then handle in the worker's `scheduled` event handler. OpenNext doesn't auto-wire these â€” **manual implementation needed**.

### 2. OpenClaw Gateway WebSocket (`gateway-ws.ts`)
Uses Node.js `http`, `crypto`, `net` modules for raw WebSocket framing. Currently disabled (`KYRA_USE_OPENCLAW=false`), but if re-enabled, it **will not work** on Cloudflare Workers. Would need rewriting to use the Workers WebSocket API.

### 3. jsdom / URL Fetching (`lib/tools/url-fetch.ts`)
jsdom is a heavy Node.js dependency. It works with `nodejs_compat` in OpenNext, but may have runtime issues with some DOM features. The `simpleFetch` fallback in the same file should work fine.

### 4. Worker Size Limits
Cloudflare Workers have a 10MB compressed limit (paid plan). Current bundle is ~15MB uncompressed / ~2.6MB gzip â€” well within limits.

### 5. Cold Starts
Workers have near-zero cold starts (vs Vercel's serverless cold starts), but the first request to a new route may be slightly slower due to JIT compilation.

### 6. Image Optimization
Vercel's built-in image optimization is not available. Consider using Cloudflare Images or a custom loader.

## Rollback

To revert to Vercel, simply:
1. Switch back to the `main` branch
2. Push to Vercel (it still has the project configured)
3. Update DNS for `kyra.conversionsystem.com` back to Vercel
